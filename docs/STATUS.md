## Phase

**v1.2 — text word-game arcade (web).** The web app was pivoted from the graph-based hop
game to a **text-only arcade of three server-authoritative word games** over the same KG
(no graph visualization): **Alchimie** (Infinite-Craft combine), **Cald sau Rece**
(Contexto hot/cold), **Lanțul Cuvintelor** (word-ladder). Backed by
`cat_de_roman_esti/wordgames/` (a shared `service.py` over the offline KG + one
`APIRouter` per game under `/api/wordgames/*`). The old graph SPA + its `/api/games`
endpoints were removed (`react-force-graph-2d` dropped; bundle 482→289 KB).

The **terminal CLI** remains the original `easy|hard` semantic-hop game (engine/graph/data
untouched). The KG itself is the v1.1 expansion below.

**v1.1 (still in place):** a fact-checked KG of **249 nodes / 417 edges / 148 puzzles**,
**pinned web deps** for reproducible deploys, and a **GitHub Actions CI** gate.

## What's built

### Game core (Python, stdlib-only)
- `engine.py` `HopGame` — load a puzzle + its category subgraph; mode `easy|hard`; easy filters
  distractors + exposes `hint_neighbors` + edge `label_ro`, hard keeps decoys + hides them;
  validates each hop is a real edge; tracks the path; **`undo()` steps back one hop** (inverse
  of `hop`); scores vs `par`; detects win.
- `graph.py` (`Node`/`Edge`/`Graph`), `data.py` (`load_fixture` offline / `load_from_client` live →
  `KgBundle`), `roedu_client.py` (vendored stdlib client, fail-closed), `cli.py` (terminal-playable).

### Knowledge-graph content
- `fixtures/kg_sample.json` (byte-identical copy in `tests/fixtures/`): **249 nodes / 417 edges
  (50 distractors) / 148 puzzles** across **8 categories** (istorie, literatura, geografie,
  personalitati, arta_cultura, stiinta, societate, limba) **+ `mixed`**. Every category fills
  **easy [2,3] and hard [4,7]**, multiple puzzles per bucket. Puzzles are **generated by
  `romania_scraper.kg`** (valid by construction) and **fact-checked** (web-verified the risky claims).
- **v1.1 expansion** (`scripts/expand_content.py` + `scripts/expansion_data.json`): +54 new
  nodes / +115 edges (incl. 8 cross-category bridges) / +72 puzzles, all **additive** (no
  existing record altered, no existing puzzle shortened) and **web-backed fact-checked**. The
  builder regenerates puzzles via the SAME direction-aware BFS the validator re-derives, so the
  expansion is GREEN by construction; restore the base + re-run to iterate.

### Web app
- `cat_de_roman_esti/web/` — FastAPI BFF: in-memory sessions, per-mode views, `/api/*`
  (health/catalog/games/hop/reset), static SPA serving, offline default + live fail-soft. The
  `cat-de-roman-dev` API key stays server-side.
- `frontend/` — React 18 + Vite + TS SPA: **force-directed semantic-network map**
  (`react-force-graph-2d`, salience-sized glowing nodes, pulsing current/target, golden hop-trail),
  `framer-motion` screen transitions + win celebration, **Web-Audio synthesized SFX** + persisted
  mute toggle, **mobile/touch tuning** (tap-to-hop, pinch-zoom, responsive menu/HUD). Builds into
  `cat_de_roman_esti/web/static/`.
- **v1.1 UX:** an **Undo** button + Backspace (server-authoritative `/api/games/{id}/undo`),
  **multi-puzzle runs** (a session tally of solved-count + cumulative score, each win counted
  once), an offline **leaderboard** (`leaderboard.ts`, localStorage best score/hops per
  category+mode + best run, shown on Menu + a "record personal" badge on Win), and **sound +
  animation polish** (undo blip, new-record sparkle).

### Hardening
- `scripts/validate_fixture.py` — stdlib CI-gate validator, **13 invariant classes** (incl. the
  **no-distractor-shortcut-below-par** check). `tests/test_fixture_invariants.py` runs it every test.

### Deploy
- Multi-stage `Dockerfile` (node build → `python:3.11-slim`, non-root, ~144 MB), `.dockerignore`,
  `docker-compose.yml`, one-command `run.sh` + `Makefile` (`run`/`dev`/`docker`).

### Docs

## Run
```
./run.sh                 # local, offline, auto-port from 8000   (or: make run)
make dev                 # vite + uvicorn --reload (hot reload)
docker build -t cat-de-roman . && docker run --rm -p 8000:8000 cat-de-roman
# live data: ROEDU_API_URL=http://localhost:8077 ROEDU_API_KEY=cat-de-roman-dev ./run.sh
```

## Integration
Consumes `kg_nodes` / `kg_edges` / `kg_puzzles` from `ro_data_server` via the vendored
`RoeduClient` (api key `cat-de-roman-dev`, scope-isolated). The producer `kg.db` + the three
`kg_*` dataapi products + the auth key live in `romania_scraper` (landed on its `main`).
